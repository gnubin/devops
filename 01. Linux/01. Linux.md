
***
[**Linux**](https://ru.wikipedia.org/wiki/Linux) - это не конкретная операционная система, а семейство ОС на базе ядра Linux, возникшего благодаря **Линусу Торвальдсу**.

В чистом Linux Торвальдса не было Bash, графического интерфейса, текстового редактора и правильнее было бы называть [**GNU](https://ru.wikipedia.org/wiki/%D0%9F%D1%80%D0%BE%D0%B5%D0%BA%D1%82_GNU)/Linux** и чтить **Ричарда Столлмана** на ровне с Линусом. 
На базе ядра **Linux** работают Android-телефоны, Веб-Сервера, Суперкомпьютеры, Бытовые приборы и так до бесконечности. Не удивлюсь, если умные японские унитазы тоже используют **Linux** и на них можно [**запустить**](https://dtf.ru/flood/2244-9-neozhidannyh-ustroistv-na-kotoryh-zapuskali-doom) Doom.

🔗 [**Linux kernel source tree**](https://github.com/torvalds/linux) - официальный репозиторий исходного кода ядра Linux.

> [!info ]  Подсказка
> Чтобы попробовать Linux  - не обязательно ставить Ubuntu основной (или даже второй) операционной системой или покупать MacBook.

## 1. Знакомство с командной строкой (Терминал)

Независимо от способа, после первого входа выполни в терминале:

```bash
sudo apt update && sudo apt upgrade -y
```

**Что это значит:**
- `sudo` — выполнить команду с правами администратора (у тебя запросят пароль)
- `apt` — менеджер пакетов Ubuntu
- `update` — обновить информацию о доступных программах
- `upgrade` — установить обновления
- `-y` — автоматически отвечать "да" на вопросы
- `&&` — выполнить вторую команду, если первая завершилась успешно
***
Терминал (или командная строка) — это текстовый интерфейс для управления компьютером. Через него можно делать всё то же, что и через графический интерфейс, но быстрее, точнее и автоматизируемо.

**Как открыть терминал в Ubuntu:**
- Горячие клавиши: `Ctrl + Alt + T`
- Или найди "Терминал" в меню приложений

Гугление занимает почти половину всего рабочего времени в работе. Поэтому начнем потихоньку учиться искать информацию из открытых источников. 

Нужно научиться:
- Открывать терминал
- Переходить между директориями
- Смотреть содержимое директории (скрытые файлы тоже)
- Узнать рабочую директорию
- Узнать текущего пользователя
- Вывести содержимое файла на экран
- Создать/скопировать/переместить/удалить директорию
- Создать/скопировать/переместить/удалить файл
- Отредактировать созданный файл (например написать туда "hello" и сохранить)

Список команд, с которыми нужно познакомиться

```bash
cd 
ls
pwd
whoami
cat
cp
mv
rm
touch
nano
```

## **Сети в Linux**

Статья про модель OSI и TCP/IP понятным языком - [https://habr.com/ru/articles/820419/](https://habr.com/ru/articles/820419/)

### Основные сетевые протоколы.  

**1. TCP (Transmission Control Protocol)**

Основные характеристики:
- Надёжная доставка – гарантирует, что данные дойдут без потерь (использует подтверждения ACK).
- Установка соединения – трёхэтапное рукопожатие (`SYN → SYN-ACK → ACK`).
- Контроль перегрузки – динамически регулирует скорость передачи
- Порядок доставки – пакеты приходят в корректной последовательности.

Где используется?
- Веб-серверы (HTTP/HTTPS)
- SSH, FTP, базы данных (PostgreSQL, MySQL)

---

**2. UDP (User Datagram Protocol)**

Основные характеристики:
- Без установки соединения – нет рукопожатия, пакеты отправляются сразу.
- Нет гарантии доставки – потерянные пакеты не пересылаются.
- Минимальные накладные расходы – меньше задержка, выше скорость.

Где используется?
- Видеостриминг (RTP, WebRTC)
- VoIP (SIP, Skype)
- DNS-запросы

---

**3. ICMP (Internet Control Message Protocol)**

Основные функции:
- Диагностика сети (`ping`, `traceroute`).
- Сообщение об ошибках (например, `Destination Unreachable`).

---

**4. DNS (Domain Name System) DNS** - фундаментальная технология современной интернет-среды, которая отвечает за хранение и обработку информации о доменных адресах. Нужен для того, чтобы соотносить айпи адреса с доменными именами.

Как работает DNS?
1. Браузер запрашивает `google.com`.
2. Рекурсивный резолвер (например, `8.8.8.8`) ищет запись:
- Кэш → Рут-серверы → TLD (.com) → Авторитативные серверы.
1. Возвращается IP-адрес (`A` или `AAAA` для IPv6).

---

**5. HTTP/HTTPS**

HTTP (HyperText Transfer Protocol)
- Работает поверх TCP.
- Методы запросов: `GET`, `POST`, `PUT`, `DELETE`.
- Коды ответов:
	- `200 OK` – успех
	- `404 Not Found` – страница не найдена
	- `500 Internal Server Error` – ошибка сервера

HTTPS (HTTP + TLS/SSL)
- Отличие от HTTP - Шифрование трафика (порт `443 вместо 80 у HTTP`).
- Сертификаты (Let's Encrypt, самоподписанные).

## Управление сервисами

**Systemd** — подсистема инициализации и управления службами в Linux. Под службами понимаются процессы, создаваемые как раз systemd (или другими системами инициализации).

**Systemctl** - утилита управления службами через команданую строку  
  
Базовый набор команд для управления службами:  
systemctl status <service> - показывает статус службы

systemctl start/stop <service> - запускает или останавливает службу

systemctl enable/disable <service> - включить/отключить автозагрузку службы

journalctl -u <service> - показать логи службы
