23. [[Как метрики попадают в Grafana? Что такое источник данных (datasource)?]]

Отличный, фундаментальный вопрос! Он позволяет понять саму суть работы Grafana. Объясню максимально просто и подробно.

### Как метрики попадают в Grafana? (Простая аналогия)

Представьте, что **Grafana — это великолепный, навороченный телевизор с крутым экраном.** А **метрики** — это телеканалы (как National Geographic или Discovery).

**Как телеканалы попадают на телевизор?** Они не появляются там сами по себе. Телевизор должен быть **настроен на прием сигнала** от спутниковой тарелки, кабельного провайдера или антенны.

**Источник данных (Datasource) в Grafana — это и есть настройка на этого "провайдера" данных.** Вы говорите телевизору: *"Вот адрес моей спутниковой тарелки (Prometheus), вот как к ней подключиться (логин, пароль, порт)".*

После этой настройки вы можете переключать каналы (создавать дашборды и графики), а телевизор (Grafana) будет в реальном времени брать "телепередачу" (метрики) у провайдера (Prometheus) и показывать вам.

---

### Технический процесс (более детально)

Процесс можно разбить на три ключевых шага:

1.  **Сбор и Хранение (Не задача Grafana):**
    *   Данные (метрики, логи) собираются и **хранятся где-то в другом месте**. Grafana **сама по себе не хранит данные** (за исключением временных аннотаций).
    *   Для метрик это обычно **Prometheus**, **InfluxDB**, **TimescaleDB** или облачные сервисы (Amazon CloudWatch, Google Cloud Monitoring).
    *   Для логов это **Loki**, **Elasticsearch**.
    *   Эти системы являются **поставщиками данных**.

2.  **Настройка подключения (Источник данных - Datasource):**
    *   В интерфейсе Grafana вы заходите в `Configuration -> Data Sources -> Add data source`.
    *   Выбираете тип вашего "провайдера" (например, Prometheus).
    *   Указываете его **адрес (URL)** (например, `http://prometheus:9090`). Это критически важный шаг. Grafana должна знать, *куда* стучаться за данными.
    *   При необходимости указываете аутентификацию (логин, пароль, токен).
    *   Нажимаете `Save & Test`. Если Grafana может подключиться к указанному адресу, вы получаете зеленую плашку "Data source is working".

    ![Data Source Configuration in Grafana](https://grafana.com/static/img/docs/add-data-source/add-prometheus-data-source-8-0.png)

3.  **Запрос и Визуализация (Создание графиков):**
    *   Теперь вы создаете новый дашборд или график.
    *   В настройках графика выбираете, к какому **источнику данных (Datasource)** он должен обращаться (например, ваш только что добавленный Prometheus).
    *   Пишете **запрос** на языке вашего источника данных.
        *   Для Prometheus это **PromQL** (например, `rate(node_cpu_seconds_total[5m])`).
        *   Для Loki это **LogQL**.
        *   Для CloudWatch это свой собственный синтаксис.
    *   Grafana **отправляет этот запрос** по указанному в Datasource адресу.
    *   Источник данных (например, Prometheus) **выполняет запрос** против своих внутренних хранилищ и **возвращает результат** (данные во временном формате) обратно в Grafana.
    *   Grafana получает эти данные и **отрисовывает** их в виде красивого графика, таблицы или иного визуального элемента.

**Итак, метрики попадают в Grafana по следующей цепочке:**
**Система сбора (Prometheus)** --(хранит данные)--> **Запрос из Grafana** --> **Источник данных (настройка подключения)** --> **Ответ от Prometheus** --> **Визуализация в Grafana**.

---

### Что такое Источник Данных (Datasource)? - Развернутое определение

**Источник данных (Datasource)** — это конфигурация внутри Grafana, которая определяет:

1.  **Тип базы данных/сервиса,** из которого будут браться данные (Prometheus, Loki, MySQL, Postgres, Elasticsearch, AWS CloudWatch и еще более 100 вариантов).
2.  **Сетевой адрес (URL)** этого сервиса. Это может быть внутренний адрес в Kubernetes (`http://prometheus-server.monitoring.svc.cluster.local:9090`) или внешний URL.
3.  **Параметры аутентификации** для доступа к этому сервису (например, логин/пароль, API-токен, IAM-роль в AWS).
4.  **Дополнительные настройки:** Таймауты, режим доступа (прямой или через браузер), версия API и т.д.

**Ключевая мысль:** Добавление источника данных **не импортирует сами данные** в Grafana. Это просто **подключение**, "провод", по которому Grafana будет делать запросы, когда это потребуется для отрисовки дашборда.

### Итог для собеседования

"**Grafana сама не хранит метрики.** Она является мощным инструментом **визуализации и анализа**. Метрики попадают в Grafana следующим образом:

1.  Сначала они собираются и сохраняются в специализированных базах данных временных рядов, таких как **Prometheus**.
2.  В Grafana настраивается **Источник Данных (Datasource)**. По сути, это конфигурационное соединение, в котором указывается *тип* (например, Prometheus) и *адрес* сервера, где хранятся данные.
3.  При создании графика на дашборде Grafana отправляет **запрос** (например, на языке PromQL) через настроенный источник данных к настоящей БД.
4.  База данных обрабатывает запрос и **возвращает результат** (сырые данные) в Grafana, которая уже и отрисовывает их в удобном для пользователя виде.

Таким образом, **Источник Данных — это мост** между движком визуализации (Grafana) и системой хранения данных. Без правильно настроенного источника данных Grafana не знает, откуда брать информацию для графиков."
