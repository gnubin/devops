#git
***
[**Git](https://ru.wikipedia.org/wiki/Git)** - программа, позволяющая организовать парелльную разработку ПО за счет изолированных копий исходного кода с возможностью сливать изменения в единое целое после готовности, не мешая другим участникам разработки. Сохраняет всю историю изменений, позволяя откатывать изменения на предыдущие состояния кода.

Разработан Линусом Торвальдсом в 2005 году для удобной совместной разработки Linux. Историю создания можно почитать на [Википедии](https://ru.wikipedia.org/wiki/Git). Согласно опросу разработчиков [Stack Overflow 2022](https://survey.stackoverflow.co/2022#section-version-control-version-control-systems), Git – самая популярная система контроля версий, которую используют более 93,87% разработчиков

В повседневной работе DevOps-инженера используется для хранения конфигураций всего на свете, хранения скриптов, а также совместной работы над всем этим. В том числе DevOps-инженер тесно интегрирован с командами разработки и так или иначе взаимодействует с исходным кодом разрабатываемых программ.

Git и [**GitLab**](https://ru.wikipedia.org/wiki/GitLab) не одно и тоже, однако это плоть от плоти, так как GitLab как и [**BitBucket**](https://ru.wikipedia.org/wiki/Bitbucket) в основе своей имеют git + веб-интерфейс + куча всего сверх того. На основе Git также работает [**GitHub](https://ru.wikipedia.org/wiki/GitHub)** и другие подобные решения.
